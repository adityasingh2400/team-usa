# Compliance Notes

## NIL And Athlete Matching

The risky version of a "characteristic matcher" is: user enters traits and gets
a specific athlete match. That creates a name/image/likeness problem and also
invites implied performance claims.

The safer version is: user enters traits and gets aggregate historical
archetypes, sport clusters, or event families.

Allowed output pattern:

- "Your inputs could align with historical endurance archetypes seen in Team USA
  distance events."
- "This profile may be associated with sports where balance, leverage, and
  repeated power output are commonly represented."
- "For Paralympic pathways, classification is sport-specific; this tool explains
  classification concepts but does not infer disability status or eligibility."

Avoid:

- "You are most like [named athlete]."
- Photos, video, generated likenesses, or silhouette mimicry of real athletes.
- "This body type means you should/will be good at this sport."
- Individual athlete cards in public UI.
- Finish times or specific scoring results.

## Core Hackathon Constraints

- Gemini API is mandatory.
- Google Cloud deployment is mandatory.
- Public Team USA data only.
- Larger international Olympic/Paralympic datasets must be filtered to Team USA
  scope before use in the product.
- Finish placement and medals are permitted.
- Finish times and specific scoring results are prohibited.
- Team USA multimedia that shows athlete NIL is prohibited.
- IOC/USOPC/LA28 protected visual IP must not appear in the project UI.
- Generative media must be animation only and must not resemble real people.
- Public repo must use Apache License 2.0.

## Data Handling Rule For This Repo

Because the FAQ says only Google Cloud generative AI tools may process Team USA
data, keep this repository focused on:

- Source URLs.
- Terms/licensing notes.
- Schemas and field allowlists.
- Deterministic ingestion code.
- Aggregated outputs generated by the Google Cloud pipeline.

Do not paste athlete-level rows into prompts or use non-Google GenAI to analyze
them.
